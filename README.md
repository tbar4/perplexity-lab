# trevor-barnes-thesis

Public site: **https://tbar4.github.io/perplexity-lab/**

> Notes, experiments, and artifacts toward a master's in Space Studies at the University of North Dakota.

The site combines a **Zettelkasten** of typed-link notes with a multi-format **artifacts catalog** (apps, documents, images, notebooks, datasets, videos). Everything is open and reachable at a stable URL.

> The GitHub repo is still named `perplexity-lab` to preserve URLs; the site brand is "Trevor Barnes · Thesis".

## Zettelkasten

### Note format

```markdown
---
id: 202605172100-some-slug          # YYYYMMDDHHMM-slug
title: "Note title"
created: 2026-05-17
updated: 2026-05-17
type: permanent | literature | fleeting | hub | source
status: seedling | growing | evergreen
tags: [tag1, tag2]
thesis_chapter: 3                   # optional
visibility: public | draft           # draft = excluded from build
links:
  references: [other-id]
  elaborates: [other-id]
  supports: [other-id]
  contradicts: [other-id]
  example_of: [other-id]
  defines: [other-id]
  questions: [other-id]
  derived_from: [other-id]
---

Body markdown. Use [[type::Target Note]] for typed inline links, or
plain [[Target Note]] for an associative "related" edge.
```

### Typed link relationships

| Type | Cluster | Inverse (shown in backlinks) |
| --- | --- | --- |
| `references` | evidential | referenced by |
| `supports` | evidential | supported by |
| `contradicts` | evidential | contradicted by |
| `example_of` | evidential | has example |
| `elaborates` | structural | elaborated by |
| `defines` | structural | term used in |
| `derived_from` | structural | source of |
| `questions` | interrogative | questioned by |
| `related` | associative | related to |

Clusters drive the graph view's edge coloring: evidential edges are cyan, structural violet, interrogative amber, associative grey.

### Adding a note manually

1. Drop a new markdown file in `notes_src/` named `YYYYMMDDHHMM-slug.md`.
2. Run `python3 scripts/build_notes.py` to verify it renders and its links resolve.
3. Commit and push — the Pages workflow re-runs the build on deploy.

The easier path is to ask Perplexity Computer to publish it via the `publish-to-lab` skill.

## Artifacts catalog

The catalog supports seven artifact types: `note`, `app`, `doc`, `image`, `notebook`, `dataset`, `video`. Each lives under `apps/<id>/` and is listed in `apps/manifest.json`. Notes have a slim manifest entry pointing at their built HTML page so the unified search on the landing page covers everything.
